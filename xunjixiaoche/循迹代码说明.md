# 循迹代码修改说明

## 修改概述

已按照参考PID的循迹逻辑重写了循迹代码，主要修改了以下文件：
- `code/xunji.c` - 循迹逻辑
- `code/xunji.h` - 循迹头文件
- `code/pid.c` - PID控制器

## 核心修改

### 1. 循迹位置计算（xunji.c）

**原来的错误逻辑：**
- 使用权重累加（-8, -4, -2, -1, 1, 2, 4, 8）
- 将累加结果直接作为角度目标（0°, 90°, -90°等）
- 这与串级PID的位置控制逻辑不匹配

**新的正确逻辑：**
```c
// 传感器位置定义：0-7（对应8个传感器）
// 中心位置：3.5
// 计算方法：加权平均位置
position = Σ(sensor_position[i]) / Σ(检测到黑线的传感器数量)
```

**位置含义：**
- `weighted_value = 0.0` → 黑线在最左边（传感器0）
- `weighted_value = 3.5` → 黑线在中心（目标位置）
- `weighted_value = 7.0` → 黑线在最右边（传感器7）

### 2. PID控制逻辑（pid.c）

**串级PID结构：**

```
外环（位置环）：
  输入：current_position（黑线位置 0-7）
  目标：target_position = 3.5（中心位置）
  输出：position_correction（速度修正量）

内环（速度环）：
  输入：left_speed_target, right_speed_target
  输出：MOTOl, MOTOr（电机PWM）
```

**控制逻辑：**
```c
// 位置偏差
error = current_position - target_position

// 当 weighted_value > 3.5（黑线在右边）
// error > 0 → position_correction > 0
// left_speed_target = base_speed - correction（左轮减速）
// right_speed_target = base_speed + correction（右轮加速）
// 结果：左转

// 当 weighted_value < 3.5（黑线在左边）
// error < 0 → position_correction < 0
// left_speed_target = base_speed + correction（左轮加速）
// right_speed_target = base_speed - correction（右轮减速）
// 结果：右转
```

### 3. PID参数调整

**位置环PID参数（需要根据实际调试）：**
```c
Kp_pos = 20.0f;   // 比例系数（建议范围：15-25）
Ki_pos = 0.0f;    // 积分系数（建议初始为0，稳定后可加0.1-0.5）
Kd_pos = 10.0f;   // 微分系数（建议范围：5-15）
gyro_kd = 100.0f; // 陀螺仪角速度系数（建议范围：50-200）

pos_out_max = 200.0f;   // 位置环输出上限
pos_out_min = -200.0f;  // 位置环输出下限
```

**陀螺仪的作用：**
- **提前预测转向**：陀螺仪检测车身旋转角速度，比位置传感器反应更快
- **抑制震荡**：相当于额外的微分项，能快速抑制车身摆动
- **提高稳定性**：在高速或急转弯时效果更明显
- **工作原理**：当车身右转时（GyrZ > 0），减小右转力度；当车身左转时（GyrZ < 0），减小左转力度

**速度环PID参数（保持原有）：**
```c
Kp_l = 2.0f, Ki_l = 0.15f;  // 左轮
Kp_r = 2.0f, Ki_r = 0.15f;  // 右轮
Kd_l = 0.0f, Kd_r = 0.0f;
```

## 调试建议

### 第一步：验证传感器数据
1. 在OLED上显示 `weighted_value` 的值
2. 将小车放在黑线上，观察：
   - 黑线在最左边 → weighted_value ≈ 0-1
   - 黑线在中心 → weighted_value ≈ 3-4
   - 黑线在最右边 → weighted_value ≈ 6-7

### 第二步：调试位置环PID
1. **先调Kp（比例）：**
   - 从 Kp=10 开始
   - 如果反应太慢，增大Kp（15, 20, 25...）
   - 如果震荡严重，减小Kp
   - 目标：小车能跟随黑线，但可能有震荡

2. **再调Kd（微分）：**
   - 从 Kd=5 开始
   - 增大Kd可以抑制震荡
   - 如果反应变慢，减小Kd
   - 目标：减少震荡，提高稳定性

3. **调整陀螺仪系数（gyro_kd）：**
   - 从 gyro_kd=100 开始
   - 如果高速时震荡严重，增大gyro_kd（120, 150, 180...）
   - 如果转弯反应迟钝，减小gyro_kd（80, 60, 50...）
   - 如果陀螺仪数据不准确，可以设为0禁用
   - 目标：高速时更稳定，转弯更流畅

4. **最后调Ki（积分）：**
   - 初始保持 Ki=0
   - 如果有稳态误差（始终偏离中心），加小量Ki（0.1, 0.2...）
   - Ki过大会导致震荡

### 第三步：调整速度参数
```c
base_speed = 100;  // 基础速度（可根据实际调整）
```
- 速度太快：循迹不稳定，容易冲出黑线
- 速度太慢：反应迟钝

### 第四步：调整动态降速
```c
i_speed = base_speed - (fmin(fabs(current_position - target_position), 3.5f) / 3.5f) * 50;
```
- 最后的 `50` 是降速幅度
- 增大这个值：转弯时降速更多（更稳定但更慢）
- 减小这个值：转弯时降速更少（更快但可能不稳定）

## 常见问题

### 问题1：小车不动或乱动
- 检查 `star_car` 是否为1（按键启动）
- 检查传感器是否正常读取（Digtal值）
- 检查电机方向是否正确

### 问题2：小车左右震荡
- Kp太大，减小Kp
- 增大Kd抑制震荡
- 检查传感器安装高度是否合适

### 问题3：小车偏离黑线
- 检查传感器数据是否正确
- 检查位置计算是否准确
- 调整Kp，增强响应

### 问题4：转弯时冲出黑线
- 减小base_speed
- 增大动态降速幅度（50 → 80）
- 增大Kd

## 代码流程

```
主循环（main.c）
  ↓
gw_get_value()          // 读取8路灰度传感器（I2C）
  ↓
track_line()            // 计算黑线位置，设置target_position
  ↓
TIM5中断（定时调用）
  ↓
control_speed()         // 串级PID控制
  ├─ 位置环：计算position_correction
  ├─ 速度环：计算left/right_speed_target
  └─ 输出：set_speed(MOTOl, MOTOr)
```

## 注意事项

1. **传感器方向**：确认传感器0在左边，传感器7在右边
2. **电机方向**：确认position_correction的正负与转向一致
3. **参数保存**：调试好的参数记得保存，避免重新调试
4. **安全测试**：先在地面测试，确认方向正确后再上赛道
