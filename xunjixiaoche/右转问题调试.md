# 右转问题调试指南

## 问题描述
右转时小车会先往右转一点就马上向左转了。

## 可能的原因

### 1. 位置历史被污染
如果位置历史中混入了左转的数据，平均值会偏左，导致误判。

### 2. 判断阈值问题
原来的判断条件 `avg_position > 3.5f` 太接近中心，容易误判。

### 3. 速度叠加方向问题
检查速度叠加的正负号是否正确。

## 调试步骤

### 第1步：显示关键数据

在 `main.c` 中添加：

```c
extern float weighted_value;
extern uint8_t line_lost;
extern float get_average_position(void);  // 需要在xunji.h中声明为extern

// 在主循环中
sprintf((char *)send, "W:%.1f A:%.1f L:%d",

OLED_ShowString(0, 0, (uint8_t *)send, 16, 1);
```


**异常情况（会左转）：**
```

```

### 第3步：检查速度叠加方向

在 `pid.c` 的 `control_speed()` 中添加调试：

```c
// 在计算速度目标后
sprintf((char *)debug, "Pos:%.1f Cor:%.0f", current_position, position_correction);
// 通过串口或OLED输出
```

**正确的逻辑：**
- `current_position = 7.0`（右边）
- `error = 7.0 - 3.5 = 3.5`（正值）
- `position_correction > 0`（正值）
- `left_speed = base + correction`（左轮加速）
- `right_speed = base - correction`（右轮减速）
- **结果：右转** ✓

## 修改说明

我已经修改了判断逻辑：

### 原来的问题：
```c
if(avg_position < 3.5f && avg_position != 0)  // 太接近中心
    return 0.0f;
else if(avg_position > 3.5f && avg_position != 0)  // 太接近中心
    return 7.0f;
```

**问题：**
- `avg_position = 3.6` 时会右转（返回7.0）
- 但如果历史中有几个左转数据，平均值可能变成 3.4
- 然后就会左转（返回0.0）

### 修改后：
```c
if(avg_position < 3.0f)  // 明确在左边
    return 0.0f;
else if(avg_position > 4.0f)  // 明确在右边
    return 7.0f;
else  // 中间区域（3.0-4.0），用last_valid_position判断
{
    if(last_valid_position < 3.5f)
        return 0.0f;
    else if(last_valid_position > 3.5f)
        return 7.0f;
    else
        return 3.5f;
}
```

**优势：**
- 只有明确在左边（< 3.0）才左转
- 只有明确在右边（> 4.0）才右转
- 中间区域用最后一次有效位置判断

## 如果还是有问题

### 方案1：增大判断阈值
```c
if(avg_position < 2.5f)  // 更明确在左边
    return 0.0f;
else if(avg_position > 4.5f)  // 更明确在右边
    return 7.0f;
```

### 方案2：只用最近的位置
```c
// 不用平均值，只用最后一次有效位置
if(last_valid_position < 3.5f)
    return 0.0f;
else if(last_valid_position > 3.5f)
    return 7.0f;
```

### 方案3：清空历史记录
在启动时清空位置历史：

```c
// 在按键启动时
if(star_car == 1)
{
    // 清空位置历史
    for(int i = 0; i < POSITION_HISTORY_SIZE; i++)
    {
        position_history[i] = 3.5f;
    }
}
```

## 快速测试

### 测试1：强制右转
临时修改代码，丢线时强制右转：

```c
// 在calculate_line_position()中
if(line_lost)
{
    return 7.0f;  // 强制右转
}
```

如果这样能正常右转，说明问题在判断逻辑。

### 测试2：检查速度方向
临时修改代码，强制左右轮速度：

```c
// 在control_speed()中
left_speed_target = 100;   // 左轮固定速度
right_speed_target = 50;   // 右轮减速
// 应该右转
```

如果这样不能右转，说明问题在电机控制。

## 总结

最可能的原因是：
1. **位置历史被污染**：历史中混入了左转数据
2. **判断阈值太小**：3.5的阈值太接近中心

建议：
1. 先用OLED显示 `weighted_value` 和 `avg_position`
2. 观察右转时这两个值的变化
3. 如果 `avg_position` 没有超过4.0，说明历史被污染
4. 可以尝试增大阈值或清空历史记录
