# 直角弯和丢线处理说明

## 问题描述

在转直角弯时，小车的8路灰度传感器会出现**全部扫不到黑线**的情况（丢线），导致小车不知道该往哪个方向转，可能会冲出赛道。

## 解决方案

### 1. 丢线检测

在 `xunji.c` 中添加了丢线检测逻辑：

```c
static uint8_t line_lost = 0;  // 丢线标志：0=正常，1=丢线

// 在计算位置时
if(black_count > 0)
{
    // 检测到黑线
    line_lost = 0;
    return position;
}
else
{
    // 未检测到黑线（丢线）
    line_lost = 1;
    // 根据上次位置继续转向
}
```

### 2. 丢线时的转向策略

**核心思想**：根据丢线前黑线的位置，判断应该往哪个方向继续转。

```c
if(last_valid_position < 3.5f)
{
    // 上次黑线在左边，说明正在左转
    // 继续左转寻找黑线
    return 0.0f;  // 返回最左边位置，产生大的左转偏差
}
else if(last_valid_position > 3.5f)
{
    // 上次黑线在右边，说明正在右转
    // 继续右转寻找黑线
    return 7.0f;  // 返回最右边位置，产生大的右转偏差
}
```

**工作原理**：

| 情况 | last_valid_position | 返回值 | 偏差 | 动作 |
|------|---------------------|--------|------|------|
| 左转丢线 | < 3.5 | 0.0 | 0 - 3.5 = -3.5 | 大幅左转 |
| 右转丢线 | > 3.5 | 7.0 | 7.0 - 3.5 = +3.5 | 大幅右转 |
| 直行丢线 | ≈ 3.5 | 3.5 | 0 | 保持直行 |

### 3. 丢线时降速

在 `pid.c` 中添加了丢线时的降速逻辑：

```c
if(is_line_lost())
{
    // 丢线时降低速度到60%，提高直角弯通过率
    i_speed = base_speed * 0.6f;
}
else
{
    // 正常循迹，根据偏差动态调速
    i_speed = base_speed - (fmin(fabs(current_position - target_position), 3.5f) / 3.5f) * 50;
}
```

**为什么要降速？**
- 速度太快，转弯半径大，容易冲出赛道
- 降速后有更多时间重新找到黑线
- 提高直角弯的通过成功率

## 直角弯通过过程

### 左直角弯示例：

```
1. 进入弯道前：
   传感器：[1,1,1,0,0,1,1,1]
   位置：3.5（中心）
   状态：正常循迹

2. 开始左转：
   传感器：[1,1,0,0,1,1,1,1]
   位置：2.5（偏左）
   状态：正常循迹，开始左转

3. 丢线（关键时刻）：
   传感器：[1,1,1,1,1,1,1,1]（全白）
   位置：0.0（强制最左）
   状态：丢线，继续大幅左转
   速度：降低到60%

4. 重新找到黑线：
   传感器：[0,0,1,1,1,1,1,1]
   位置：0.5（左边）
   状态：恢复正常，继续循迹
```

## 参数调整

### 1. 丢线时的降速比例

```c
// 在 pid.c 中
i_speed = base_speed * 0.6f;  // 降低到60%
```

**调整建议：**
- 如果直角弯转不过来（冲出赛道）：减小比例（0.5, 0.4...）
- 如果直角弯转得太慢：增大比例（0.7, 0.8...）
- 建议范围：0.5 - 0.8

### 2. 基础速度

```c
uint16_t base_speed = 100;
```

**调整建议：**
- 先用较低速度（80-100）调试直角弯
- 确认能稳定通过后，再逐步提高速度
- 速度越高，直角弯越难通过

## 调试技巧

### 1. 在OLED上显示丢线状态

在 `main.c` 中添加：

```c
extern uint8_t is_line_lost(void);

// 在主循环中
sprintf((char *)send, "Lost:%d Pos:%.1f  ", is_line_lost(), weighted_value);
OLED_ShowString(0, 0, (uint8_t *)send, 16, 1);
```

**观察现象：**
- 直线时：Lost=0, Pos=3.0-4.0
- 弯道时：Lost=0, Pos=0.0-7.0
- 丢线时：Lost=1, Pos=0.0或7.0

### 2. 测试直角弯

**测试步骤：**
1. 将小车放在直角弯前
2. 启动小车，观察通过情况
3. 记录是否成功通过

**常见问题：**

| 现象 | 原因 | 解决方法 |
|------|------|----------|
| 冲出赛道 | 速度太快或转向不足 | 减小base_speed或降速比例 |
| 转弯太慢，停在弯道 | 速度太慢或转向过度 | 增大base_speed或降速比例 |
| 在弯道震荡 | PID参数不合适 | 调整Kp、Kd |
| 找不到黑线 | 传感器高度不对 | 调整传感器安装高度 |

## 进阶优化（可选）

### 1. 检测直角弯类型

可以根据传感器状态判断是左直角还是右直角：

```c
// 在丢线前，检测哪些传感器触发
if(sensor[0] == 0 || sensor[1] == 0 || sensor[2] == 0)
{
    // 左侧传感器触发多，可能是左直角
}
else if(sensor[5] == 0 || sensor[6] == 0 || sensor[7] == 0)
{
    // 右侧传感器触发多，可能是右直角
}
```

### 2. 丢线计时

如果丢线时间过长（超过一定时间），可能是真的跑出赛道了：

```c
static uint32_t lost_time = 0;

if(line_lost)
{
    lost_time++;
    if(lost_time > 100)  // 丢线超过100个周期
    {
        // 可能跑出赛道，停车
        star_car = 0;
    }
}
else
{
    lost_time = 0;
}
```

### 3. 多级降速

根据偏差程度分级降速：

```c
if(is_line_lost())
{
    i_speed = base_speed * 0.5f;  // 丢线：降到50%
}
else if(fabs(current_position - target_position) > 2.0f)
{
    i_speed = base_speed * 0.7f;  // 大偏差：降到70%
}
else
{
    i_speed = base_speed;  // 小偏差：全速
}
```

## 与参考代码的对比

参考PID代码使用16路传感器，覆盖范围更大，不容易丢线。你的8路传感器覆盖范围较小，在直角弯时更容易丢线，所以需要更强的丢线处理逻辑。

## 总结

**丢线处理的核心思想：**
1. **检测丢线**：所有传感器都检测不到黑线
2. **记忆方向**：根据丢线前的位置，判断应该往哪边转
3. **继续转向**：返回极值位置（0或7），产生大的偏差，快速转向
4. **降低速度**：丢线时降速，提高转弯成功率
5. **重新找线**：持续转向直到重新找到黑线

**调试建议：**
- 先用低速（base_speed=80）测试直角弯
- 确认能稳定通过后，再逐步提高速度
- 在OLED上显示丢线状态，方便调试
- 如果还是转不过来，可以进一步降低丢线时的速度比例
