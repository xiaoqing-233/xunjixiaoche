# 陀螺仪在循迹中的应用说明

## 为什么要使用陀螺仪？

在循迹小车中，陀螺仪（IMU）的Z轴角速度（GyrZ）可以显著提升控制性能：

### 1. 提前预测转向
- **位置传感器**：只能检测到"黑线在哪里"（滞后信息）
- **陀螺仪**：能检测到"车身正在以多快的速度旋转"（实时信息）
- **优势**：陀螺仪能提前预知车身的转向趋势，比位置传感器反应更快

### 2. 抑制震荡
- 相当于在PID控制中增加了一个**额外的微分项**
- 当车身开始摆动时，陀螺仪立即检测到角速度变化
- 控制器可以快速施加反向力矩，抑制震荡

### 3. 高速稳定性
- 速度越快，车身惯性越大，越容易震荡
- 陀螺仪的快速响应能在高速时保持稳定
- 特别适合竞速型循迹小车

## 工作原理

### 控制逻辑

```c
// 位置环PID输出
output = Kp * error + Ki * Σerror + Kd * Δerror - gyro_kd * GyrZ
```

**陀螺仪项的作用：**

| 情况 | GyrZ值 | gyro_kd * GyrZ | 效果 |
|------|--------|----------------|------|
| 车身右转 | > 0 | > 0 | 减去正值 → 减小右转力度 |
| 车身左转 | < 0 | < 0 | 减去负值（相当于加）→ 减小左转力度 |
| 车身直行 | ≈ 0 | ≈ 0 | 无影响 |

**核心思想**：当车身已经在转向时，减小继续转向的力度，防止过度转向。

## 参数调试

### gyro_kd 的调试方法

1. **初始值：100**
   ```c
   float gyro_kd = 100.0f;
   ```

2. **观察现象，调整参数：**

   | 现象 | 原因 | 调整方向 |
   |------|------|----------|
   | 高速时左右摆动严重 | gyro_kd太小，抑制不足 | 增大（120, 150, 180...） |
   | 转弯反应迟钝，转不过来 | gyro_kd太大，过度抑制 | 减小（80, 60, 50...） |
   | 低速稳定，高速不稳 | gyro_kd偏小 | 增大 |
   | 转弯时冲出黑线 | gyro_kd偏大或Kp太大 | 减小gyro_kd或Kp |

3. **建议范围：**
   - **低速循迹**（< 0.5m/s）：50-100
   - **中速循迹**（0.5-1m/s）：100-150
   - **高速循迹**（> 1m/s）：150-200

### 调试顺序

```
1. 先不用陀螺仪（gyro_kd = 0）
   ↓
   调好基础的 Kp, Kd, Ki
   ↓
2. 加入陀螺仪（gyro_kd = 100）
   ↓
   观察高速时的表现
   ↓
3. 根据现象微调 gyro_kd
```

## 陀螺仪数据检查

### 1. 验证陀螺仪是否正常工作

在OLED上显示GyrZ的值：

```c
// 在main.c的主循环中
sprintf((char *)send, "GyrZ: %.2f  ", GyrZ);
OLED_ShowString(0, 0, (uint8_t *)send, 16, 1);
```

**正常现象：**
- 小车静止时：GyrZ ≈ 0（可能有小的漂移，±5以内正常）
- 手动左转小车：GyrZ < 0（负值）
- 手动右转小车：GyrZ > 0（正值）
- 转动越快，GyrZ绝对值越大

**异常现象：**
- GyrZ始终为0 → 陀螺仪未初始化或通信失败
- GyrZ值跳变很大 → 陀螺仪数据不稳定
- 转向方向与GyrZ符号相反 → 陀螺仪安装方向错误

### 2. 陀螺仪零点校准

JY901S陀螺仪需要在小车静止时进行零点校准：

```c
// 在main.c的初始化部分
JY901S_ZeroCalibration();  // 已经调用
```

**注意**：校准时小车必须完全静止，放在平稳的地面上。

## 常见问题

### Q1: 加入陀螺仪后反而更不稳定了？

**可能原因：**
1. gyro_kd设置过大，过度抑制了正常转向
2. 陀螺仪数据有噪声或漂移
3. 陀螺仪未正确校准

**解决方法：**
1. 减小gyro_kd，从50开始重新调试
2. 检查陀螺仪数据是否正常（见上文）
3. 重新进行零点校准

### Q2: 不用陀螺仪可以吗？

**可以！** 如果你的小车：
- 速度较慢（< 0.5m/s）
- 赛道比较简单（没有急转弯）
- 已经调好了Kp、Kd，效果不错

那么可以设置 `gyro_kd = 0` 禁用陀螺仪。

### Q3: 陀螺仪的效果什么时候最明显？

**最明显的场景：**
1. **高速循迹**：速度越快，陀螺仪的优势越明显
2. **急转弯**：快速转向时，陀螺仪能快速抑制过冲
3. **S型弯道**：连续转向时，陀螺仪能减少震荡

**效果不明显的场景：**
1. 低速直线循迹
2. 赛道很简单，几乎没有转弯

### Q4: 陀螺仪方向反了怎么办？

如果发现：
- 右转时GyrZ < 0（应该 > 0）
- 左转时GyrZ > 0（应该 < 0）

**解决方法：**
```c
// 在PositionPID_Calculate函数中
float gyro_term = gyro_kd * (-GyrZ);  // 加负号反转方向
```

或者在陀螺仪读取时反转：
```c
// 在jy901.c中读取数据时
GyrZ = -GyrZ;  // 反转方向
```

## 与参考代码的对应

参考PID代码中的陀螺仪使用：

```c
// 参考pid/User/PID_Controller.c 第149行
float gyro_term = controller->param.gyro_kd * LSE6DSR_data.gz_rads;
```

你的代码中：

```c
// xunjixiaoche/code/pid.c
extern float GyrZ;
float gyro_term = gyro_kd * GyrZ;
```

**区别：**
- 参考代码使用LSM6DSR陀螺仪（gz_rads是弧度/秒）
- 你的代码使用JY901S陀螺仪（GyrZ是度/秒）
- 单位不同，所以gyro_kd的数值范围也不同

## 总结

**陀螺仪的优势：**
- ✅ 提高高速稳定性
- ✅ 减少震荡
- ✅ 提前预测转向
- ✅ 改善急转弯表现

**何时使用陀螺仪：**
- ✅ 追求高速循迹
- ✅ 赛道有急转弯
- ✅ 想要更稳定的控制

**何时不用陀螺仪：**
- ❌ 低速循迹已经很稳定
- ❌ 陀螺仪数据不准确
- ❌ 赛道很简单

**建议：**
先调好基础的PID参数（Kp, Kd），再加入陀螺仪进行优化。如果陀螺仪效果不好，可以随时设置 `gyro_kd = 0` 禁用。
